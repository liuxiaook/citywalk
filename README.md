
# 东城边走边听｜西总布胡同 H5 MVP

一个基于 **H5 + 地点讲解 + 语音播报** 的城市漫游产品原型。  
以 **北京东城区西总布胡同** 为起点，通过“边走边听”的方式，把胡同里的历史、人物与城市更新故事，用更轻量、更有陪伴感的方式讲给用户听。

## 项目简介

很多人在旅行、citywalk 或陪朋友散步时，都会遇到这样的情况：

走到一个胡同、小街区、老建筑旁边，觉得这里好像有故事，但：

- 没有现场讲解
- 请导游要花钱
- 临时搜索资料太碎
- 传统景区导览又太重，不适合边走边听

所以，这个项目尝试解决一个很具体的问题：

**当用户走到某个地点时，能不能通过一个轻量 H5，直接听到这个地方的讲解？**

它不想做成传统“景区讲解工具”，而更像一个：

**懂本地故事、会陪用户边走边听的城市漫游助手。**

当前 MVP 先聚焦在：

**北京东城区 · 西总布胡同**

---

## 产品定位

本项目定位为：

**城市漫游型地点讲解 H5 原型**

它不是“景点大全”，而是一个更轻、更近、更有在地感的版本：

- 更适合 citywalk
- 更适合胡同散步
- 更适合自由行用户
- 更适合边走边听，而不是站着看说明牌

这个项目更关注的是：

- 地点讲解是否好听
- 内容是否像“朋友陪逛”
- 用户是否愿意继续听下去

---

## 当前版本包含什么

当前仓库版本是一个 **单页 H5 MVP 原型**，主要包含以下内容：

### 1. 路线首页
展示产品定位、路线主题和体验入口。

### 2. 路线预览区
通过简化地图感设计，呈现首批路线和点位分布。

### 3. 点位讲解卡片
当前已纳入的 5 个首批点位包括：

- 西总布胡同西口
- 9号院（李济深、李宗仁相关）
- 21号院（费正清相关）
- 74号（董希文《开国大典》画室相关）
- 更新观察点（“不停车胡同”）

### 4. 浏览器语音播报
点击按钮后，可直接触发浏览器内置语音播报，用来模拟真实讲解体验。

### 5. 悬浮播放器
用于模拟未来“边走边听”的持续播放状态。

## 功能亮点

- 轻量 H5 形式，便于快速部署与分享
- 点位式内容结构，适合路线型体验
- 语音播报增强“陪逛感”
- 讲解风格更偏口语化，不是传统导游词
- 优先验证真实体验，而不是先堆复杂功能

---

## 讲解风格

这个项目最重要的不是“地图”，而是“怎么讲”。

内容风格尽量避免：

- 百度百科式表达
- 景区导游腔
- 过度学术化叙述

而更偏向：

- 像懂东城的朋友在陪你逛
- 用户在路上听得进去
- 一边听，一边能联想到眼前的环境
- 不只是解释历史，也解释“现在为什么会变成这样”

---

## 为什么选择西总布胡同

西总布胡同适合作为第一条路线，原因包括：

- 胡同气质明确，步行体验强
- 同时具备历史、人物、艺术、更新等多层内容
- 既能讲“过去”，也能讲“现在”
- 周边街区具有自然延伸空间
- 适合作为东城区路线型产品的 MVP 起点

这个项目不想一上来讲“全北京”，而是先讲活一条线。

## 注意事项

当前版本语音能力依赖浏览器自带的 `speechSynthesis`，因此会有以下差异：

* 不同浏览器的中文发音不同
* 某些浏览器首次播放需要手动点击
* 浏览器静音或系统静音会导致无声
* 某些设备上语音体验较弱

因此目前的播报更适合作为：

**功能验证 / 原型体验**

而不是最终商用音频效果。
